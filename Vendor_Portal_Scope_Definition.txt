OVERVIEW
--------
This document defines the functional boundary of the BRM Vendor Portal V.1.00
as currently simulated on SAP BTP (Cloud Foundry). It distinguishes between
features that are within the current project scope (built and demonstrated) and
items that are explicitly deferred, out of scope, or candidates for a future
release / Phase 2 development.

User roles covered by this portal:
  - Vendor       : Registered supplier (PT Maju Bersama, CV Sukses Mandiri)
  - BRM          : Procurement/Finance staff (Ahmad Rizki, Siti Rahma)
  - Approver     : Senior approver with read + approve authority
  (All roles managed via SAP IAS single sign-on in the target architecture)

SECTION 2 — OUT-OF-SCOPE / FUTURE DEVELOPMENT BACKLOG
================================================================================

The items below are NOT included in V.1.00. They represent logical next phases
for a production deployment, organized by development priority tier.

────────────────────────────────────────────────────────────────────────────────
2.1  TIER 1 — HIGH PRIORITY (Prerequisite for Production Go-Live)
────────────────────────────────────────────────────────────────────────────────

  [OUT]  REAL SAP S/4HANA INTEGRATION
         Live OData/API calls replacing all mock data:
         · API_SUPPLIERINVOICE_PROCESS_SRV — actual invoice posting to SAP
         · API_BUSINESS_PARTNER — live vendor master data read
         · A_PurchaseOrder — live PO retrieval for invoice form
         · I_CompanyCode, I_Currency, I_PaymentTerms — live master data
         · SAP BPA Down Payment workflow API — actual DPR processing
         · SAP Flexible Workflow API — actual invoice approval routing
         Currently all data is in-memory mock; no actual SAP system is called.

  [OUT]  REAL AUTHENTICATION & AUTHORIZATION
         · SAP Identity Authentication Service (IAS) / Identity Provisioning
           Service (IPS) integration for SSO
         · OAuth 2.0 / SAML 2.0 token-based session management
         · Role assignment managed via SAP BTP subaccount authorizations
         · Password reset, MFA enforcement, session timeout
         Currently: static username/password comparison with no token.

  [OUT]  PERSISTENT DATA LAYER
         · SAP HANA Cloud or SAP BTP PostgreSQL for portal-specific data
           (pre-invoices, quotations, RFQ submissions not stored in SAP MM)
         · CAP (Cloud Application Programming model) service layer
         · Real-time data sync with SAP S/4HANA via event mesh or direct OData
         Currently: all state is in-memory React useState, lost on page refresh.

  [OUT]  EMAIL NOTIFICATION SERVICE
         · Automated email to vendor on: invoice status change, quotation
           award/rejection, new RFQ published
         · Email to BRM on: new invoice submitted, quotation received, RFQ
           closing date approaching
         · SAP BTP Alert Notification Service or SAP Business Network messaging
         Currently: in-app notifications only, no email.

  [OUT]  FILE UPLOAD & DOCUMENT MANAGEMENT
         · Actual binary file upload for invoice attachments (PDF, images)
         · SAP Document Management Service (DMS) or BTP Object Store integration
         · Virus scan, file type validation, size limits
         Currently: attachment section only stores file name strings, no actual
         file upload capability.

  [OUT]  AUDIT LOG & CHANGE HISTORY
         · Immutable audit trail stored in database (who changed what, when)
         · Change log visible in invoice/quotation detail panel
         · SAP Audit Log Service on BTP
         Currently: no persistence; status history is not recorded.


────────────────────────────────────────────────────────────────────────────────
2.2  TIER 2 — MEDIUM PRIORITY (Phase 2 Enhancements)
────────────────────────────────────────────────────────────────────────────────

  [OUT]  VENDOR SELF-ONBOARDING
         · New vendor registration form (company name, tax ID, bank details,
           category, certifications)
         · Document upload for KYC/KYB (NPWP certificate, SIUP, TDP, PKP cert)
         · BRM review and approval workflow for new vendor activation
         · SAP Business Partner creation via API_BUSINESS_PARTNER on approval
         · Duplicate vendor detection (same NPWP / bank account)

  [OUT]  VENDOR PROFILE SELF-SERVICE CHANGE REQUEST
         · Vendor-initiated change requests for: address, bank account, contact,
           tax certificate, NPWP number
         · Four-eyes approval (BRM review + Finance approval) before SAP BP update
         · Change request status tracking (Pending / Approved / Rejected)
         · SAP S/4HANA BP change document written on approval
         Currently: profile is read-only with a static "contact BRM" notice.

  [OUT]  VENDOR RATING & PERFORMANCE SCORECARD
         · KPI-based vendor scoring per delivery: on-time delivery %, invoice
           accuracy rate, response time to RFQs, rejection rate
         · Scorecard visible to both vendor and BRM
         · Historical trend charts (quarterly performance)
         · Integration with SAP QM / Supplier Evaluation (ME61/ME63)
         · Automatic disqualification trigger if score falls below threshold

  [OUT]  MULTI-ROUND RFQ / SEALED BID / REVERSE AUCTION
         · RFQ with multiple submission rounds (initial bid + best-and-final)
         · Sealed-bid mode: vendors cannot see competitors' quotes until closing
         · Reverse auction: real-time competitive bidding with countdown timer
         · Automatic best-quote ranking and award recommendation
         Currently: single-round open RFQ only.

  [OUT]  CONTRACT MANAGEMENT
         · Framework agreements and blanket purchase orders linked to vendor
         · Contract expiry notification and renewal workflow
         · Contract-based invoice validation (price vs. contract tolerance check)
         · Integration with SAP MM Outline Agreement (ME31K / ME32K)

  [OUT]  PURCHASE ORDER ACKNOWLEDGEMENT
         · Vendor acknowledges receipt of PO and confirms delivery date
         · Delivery date change request with BRM approval
         · PO line-level partial acknowledgement
         · Integration with SAP MM PO confirmation (ME22N / EDI 855)

  [OUT]  GOODS RECEIPT / DELIVERY NOTE SUBMISSION
         · Vendor submits delivery note / ASN (Advance Shipment Notice) before
           BRM creates Goods Receipt in SAP
         · Discrepancy flagging: vendor-declared qty vs. BRM-received qty
         · Integration with SAP MM Goods Receipt (MIGO / MFBF)

  [OUT]  PAYMENT STATUS VISIBILITY
         · Vendor can see payment run dates, payment amounts, and bank transfer
           references for cleared invoices
         · Outstanding payables aging view per company code
         · Integration with SAP FI Payment Program (F110) output
         Currently: invoice lifecycle ends at "Cleared" with a clearing doc no.
         but no payment confirmation detail.

  [OUT]  CREDIT NOTE / DEBIT NOTE PROCESSING
         · Vendor submits credit note against a previously posted invoice
         · BRM reviews and posts credit note to SAP (MM credit memo, MR8M)
         · Linked to original invoice in Document Flow

  [OUT]  WITHHOLDING TAX CERTIFICATE (BUKTI POTONG) DISTRIBUTION
         · BRM uploads WHT certificates (Bukti Potong PPh) per vendor per period
         · Vendor downloads certificates from portal
         · DJP e-Bupot integration for direct digital certificate delivery


────────────────────────────────────────────────────────────────────────────────
2.3  TIER 3 — STRATEGIC / LONG-TERM
────────────────────────────────────────────────────────────────────────────────

  [OUT]  E-FAKTUR (DJP) INTEGRATION
         · Direct API integration with Direktorat Jenderal Pajak (DJP) e-Faktur
           system for real-time VAT invoice validation and submission
         · NSFP (Nomor Seri Faktur Pajak) validation against DJP database
         · Automatic e-Faktur status check (valid / cancelled / replaced)

  [OUT]  SAP ARIBA NETWORK INTEGRATION
         · Vendor onboarding via Ariba Network supplier portal
         · PO-to-invoice flip: vendor creates invoice directly from PO in Ariba
         · cXML / EDI transaction support for automated document exchange
         · Ariba Discovery for new vendor sourcing

  [OUT]  MOBILE APPLICATION (iOS / Android)
         · Native mobile app for vendor invoice submission and status tracking
         · Push notifications for invoice events
         · Offline draft capability with sync on reconnect
         Currently: responsive web layout only, no native mobile app.

  [OUT]  AI-ASSISTED INVOICE PROCESSING
         · OCR / AI extraction of invoice data from uploaded PDF to auto-fill
           form fields (SAP AI Core / Document Information Extraction service)
         · Anomaly detection: flag invoices where amount deviates significantly
           from historical PO amounts or GR values
         · Duplicate invoice detection using fuzzy matching across vendors
         · Smart payment terms recommendation based on vendor history

  [OUT]  SUPPLIER SUSTAINABILITY / ESG MODULE
         · Vendor self-declaration of ESG metrics (carbon footprint, local
           content / TKDN percentage, SME certification, ISO 14001)
         · BRM ESG procurement reporting dashboard
         · Integration with BRM Group sustainability reporting framework

  [OUT]  MULTI-LANGUAGE SUPPORT (BAHASA INDONESIA)
         · Full Bahasa Indonesia translation for vendor-facing UI
         · Language toggle per user preference
         · Localized number/date formats per locale
         Currently: English-only UI.

  [OUT]  VENDOR PORTAL ANALYTICS DASHBOARD
         · Vendor-facing spend analytics: invoiced amounts by period, company
           code, category; payment cycle time trends
         · BRM-facing procurement analytics: sourcing pipeline value, quotation
           competitiveness index, vendor response rates, invoice processing KPIs
         · Export to SAP Analytics Cloud (SAC)

  [OUT]  INTER-COMPANY INVOICE SUPPORT
         · Cross-entity invoicing between BRM group companies (BRMS ↔ CPMS etc.)
         · Automatic inter-company posting in both sending and receiving company
           codes in SAP
         · Consolidation elimination support

  [OUT]  PEPPOL / UNIVERSAL BUSINESS LANGUAGE (UBL) SUPPORT
         · Structured electronic invoice exchange conforming to PEPPOL BIS
           Billing 3.0 / UBL 2.1 standards
         · Export invoice as machine-readable XML for ERP-to-ERP exchange
         · Indonesia PEPPOL access point connectivity (post-METS/DJP readiness)
